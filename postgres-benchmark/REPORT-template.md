## 二、测试方法

### 2.1 测试矩阵

| 场景 | pgbench 参数 | 测试维度 | 持续时间 |
|------|-------------|----------|----------|
| A. 只读吞吐量 | `-S -c 10 -j 2 -T 60` | 纯读 TPS | 60s |

| H. 持续稳定性 | `-c 50 -j 4 -T 300` | 长时间稳定性 | 300s |

### 2.2 测试顺序

1. 先部署两个 PG 实例，等待 Pod 就绪
2. 先对 Mayastor 实例执行全部 8 场景
3. 再对 LocalPath 实例执行全部 8 场景
4. 串行执行，避免资源争抢

### 2.3 指标定义

| 指标 | 单位 | 说明 | 数据来源 |
|------|------|------|----------|
| TPS | tps | 每秒事务数 | pgbench 输出 `tps =` 行 |
| 平均延迟 | ms | 每事务平均延迟 | pgbench 输出 `latency average` |
| p95 延迟 | ms | 95 分位延迟 | `--aggregate-interval` 日志计算 |
| p99 延迟 | ms | 99 分位延迟 | `--aggregate-interval` 日志计算 |
| 初始化耗时 | s | pgbench -i 执行时间 | 脚本计时 |
| CPU 利用率 | % | PG Pod CPU 使用率 | `kubectl top pod` |
| 内存利用率 | % | PG Pod 内存使用率 | `kubectl top pod` |

### 4.3 数据分析方法

#### 差异百分比计算

```text
差异 (%) = (Mayastor 值 - LocalPath 值) / LocalPath 值 × 100%

正值 → Mayastor 更优（吞吐量更高或延迟更低时为正）
负值 → LocalPath 更优
```

#### 统计显著性判断

- 差异 < 5%：可视为性能相当，无显著差异
- 差异 5%-15%：存在一定差异，需结合场景分析
- 差异 > 15%：存在显著差异，可明确得出结论
